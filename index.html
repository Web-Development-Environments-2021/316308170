Main differences between Web 1.0, Web 2.0 and Web 3.0


web 1.0

web 2.0

web 3.0

Supports read only, targets on content creativity of producer

Supports both read & write, targets on content creativity of users and producers

Supports both read & write, targets on linked data sets

Companies publish content that people consume

People publish content that other people can consume, companies build platforms that let people publish content for other people

People build applications that people can interact with, companies build platforms that let people publish services by leveraging the associations between people or special content

Content is static, one direction publishing of content without any interaction between readers or publishers.

Two way communication through social networking, blogging, wikis, tagging, user generated content and video.

Artificial intelligence and the web learning the user's preferences and delivers it a personalized web experience

Differences between web versions via Use Case

Let's take buying a refrigerator as an example. In web 1.0, if we wish to buy a refrigerator via the web, we would need the accurate url of the company which sells the refrigerator, so we can access its stataic website. In web 2.0 we could use a search engine and by searching the query "buy refrigerator", we would get many possible fitting websites, of several companies, and also related forums, blogs and chats about refrigerators, so we can publish questions and read what other users though of specific models. In web 3.0, we can chat with a bot of the company that can help us choose the fit refrigerator for our needs. In addition, based on our searching history and other user patterns, the website can personalize the provided content and display what we are looking for.
